# Capítulo 07 – É Possível Melhorar: Protótipos de Interfaces

## 1. Projeto de Interface na Prática
A proposta central é que projetar interface significa projetar experiência.

Base prática:
- Aplicar boas práticas já consolidadas
- Garantir coerência entre aparência e comportamento

## 2. Protótipos em Jogos
Protótipo é representação parcial do sistema para explorar alternativas e validar decisões.

Funções principais:
- Testar ideias com baixo risco
- Comunicar escolhas de design
- Descobrir e refinar soluções

## 3. Protótipos Hi-Fi
Características:
- Alta proximidade visual com o produto final
- Usado quando o projeto está mais consolidado
- Permite avaliação detalhada de interatividade e conteúdo

## 4. Protótipos Lo-Fi
Características:
- Rápidos e descartáveis
- Foco em estrutura, fluxo e navegação
- Baixo custo de alteração

Materiais comuns:
- Papel
- Cartões
- Post-its

## 5. Experiência do Usuário
Diretrizes enfatizadas:
- Navegação intuitiva
- Redução de carga cognitiva
- Mapeamentos naturais entre intenção e ação
- Design preparado para erro e reversão
- Padronização quando necessário

## 6. Requisitos para Produção
Antes de prototipar, definir:
- Plataforma (PC, tablet, celular etc.)
- Dispositivos e resolução
- Linguagem visual (cores, fontes, ícones)
- Coerência com o projeto geral

## 7. Síntese
O capítulo reforça o uso de protótipos como etapa essencial de validação, priorizando consistência, rapidez de iteração e qualidade da experiência.
