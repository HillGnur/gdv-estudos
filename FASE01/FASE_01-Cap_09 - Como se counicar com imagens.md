# Como se Comunicar com Imagens

## 1 Projetos de Interfaces

### 1.1 Wireframes

O **design da interface visual** é um componente fundamental do design de interface. Por meio do design de interface, combinam-se o design gráfico e o design digital, organizando as informações e facilitando a interação do usuário.

Como futuro game designer, é essencial ter uma boa noção de design e dos elementos que os objetos digitais devem ter, tais como:
- Forma
- Tamanho
- Cor
- Orientação
- Textura

#### O que é Wireframe?

O termo em inglês significa **wire (fios) e frames (quadros)**. Wireframes são quadros em **preto e branco** contendo somente as linhas que compõem os elementos da interface, que mais tarde receberão o tratamento artístico adequado.

No wireframe, o designer aplicará elementos como:
- Forma
- Tamanho
- Cor
- Imagens
- Fontes
- Texturas

**Importante**: um documento de wireframe **não é o documento final** que contém o design das telas, mas sim um documento que vai orientar a sua produção.

#### Características dos Wireframes

- São os protótipos mais utilizados para produção de websites, interfaces de games e ferramentas digitais
- Cada ficha (ou tela) representa uma página ou uma parte da tarefa
- Fichas simples com desenhos de telas digitais
- Podem ser feitos à mão ou no computador
- Servem para o usuário indicar se percebe as áreas de interação e suas funções
- Funcionam como documentos de representação da **organização**, **diagramação** e **funcionamento** dos elementos de uma página web, interface de jogo ou aplicativo mobile

#### Funções do Wireframe na Prototipagem

Do ponto de vista do projetista de interfaces, o protótipo tem a função de fornecer suporte na avaliação de alternativas diferentes no design visual e de interação. O trabalho do designer é fazer escolhas entre modelos de interação e todo o projeto visual gráfico.

Os tipos de prototipagem variam de acordo com suas finalidades:
- Testar a viabilidade técnica de uma ideia
- Esclarecer requisitos que ainda ficaram vagos
- Realizar avaliações e testes de um sistema com os usuários
- Comparar uma solução de design com o restante do projeto

#### Profissionais que Utilizam Wireframes

| Profissional | Utilidade |
|--------------|-----------|
| **Arquitetos de Informação** | Apresentam e validam o futuro site, aplicativo ou interface de jogo; usam como protótipo em testes de usabilidade |
| **Designers** | Usam como base para desenvolvimento do desenho de telas, elementos visuais e pontos de interação |
| **Redatores/Roteiristas** | Preenchem a estrutura com o conteúdo que fará parte do futuro site, aplicativo ou jogo |
| **Desenvolvedores** | Tornam tudo funcionar |
| **Clientes** | Validam tudo de acordo com requisitos do projeto e necessidades de funcionamento |
| **Usuários** | Utilizam como protótipo nos testes de usabilidade |

#### Processo de Desenvolvimento

Projetos de interface gráfica, de qualquer natureza, nascem com:
1. **Análise das necessidades** do cliente ou do projeto em si
2. **Levantamento de todos os requisitos**
3. **Criação do wireframe** do projeto
4. **Desenvolvimento do design** com base no wireframe e fluxogramas de funcionamento das telas

É a partir do wireframe e dos fluxogramas que o designer começa a dar forma ao design da interface gráfica.

#### Ferramentas Recomendadas

Para construir wireframes, recomenda-se utilizar ferramentas digitais como:
- **MockFlow** (https://www.mockflow.com): ferramenta online para prototipagem de interfaces, especialmente adequada para aplicativos mobile

## 2 Metáforas de Interfaces

### Conceito de Metáfora de Interface

A expressão **metáfora de interface** refere-se à utilização de uma analogia com alguma tecnologia que o usuário já domina. É um recurso para que o usuário tenha o domínio da interação sem a necessidade de aprendizado.

**Objetivo da metáfora:**
- Permitir que o usuário conheça as funcionalidades envolvidas
- Introduzir novas formas de o usuário interagir
- Não ficar limitada à imitação do ambiente físico

### Exemplos de Metáforas

#### 1. Ação Praticável no Mundo Físico

**Arrastar arquivo para lixeira**: Uma ação praticável no mundo físico, equivalente a "jogar no lixo". O usuário já compreende intuitivamente o que vai acontecer.

#### 2. Nova Forma de Interação

**Arrastar arquivo para impressora**: Uma forma de interagir para imprimir que **não tem precedentes no ambiente físico**, mas utiliza a metáfora de "enviar para" como porta de entrada para uma funcionalidade nova.

### Modelos Familiares

Modelos familiares são aqueles nos quais o usuário reconhece atividades do seu cotidiano. Por exemplo:
- Arrastar um objeto de uma pasta a outra (digitalmente) é proporcional a mover objetos de um lugar para outro dentro do escritório
- Tirar uma folha de uma gaveta e colocar em outra fisicamente é análogo a cortar e colar digitalmente

### Vantagens das Metáforas

**Principais contribuições:**
- Possibilita ao usuário **aprender o sistema enquanto o utiliza** (aprendizagem "learning by doing")
- É mais fácil para um instrutor ensinar o novo usuário usando nomes de objetos aos quais ele já está adaptado
- Reduz a curva de aprendizado
- Torna a interface mais intuitiva

### Limitações das Metáforas

Há críticas ao uso exclusivo de metáforas:

**Principal oposição**: alguns designers projetam interfaces limitadas ao objeto físico, sem pensar em acrescentar novas funcionalidades. A metáfora torna-se restritiva em vez de expansiva.

**Perspectiva equilibrada**: A função da metáfora de interface é:
- Mapear conhecimentos familiares
- Fazer o usuário interagir com novos conhecimentos de maneira intuitiva
- Servir como porta de entrada, não como limitação

## 3 Usabilidade e Affordance

### Conceito de Usabilidade

**Falar de usabilidade é falar indiretamente sobre eficiência.**

Um sistema é eficiente quando o usuário:
- Completa as tarefas propostas em tempo viável
- Sem desconforto
- Com o menor número possível de erros
- Com satisfação

#### Perspectiva do Design

A diferença principal está em:
- Projetar um produto **para a perspectiva do usuário** (design centrado no usuário)
- Versus projetar um produto apenas para que ele **realize uma função** (design centrado na tarefa)

### Definição de Usabilidade

De acordo com Benyon (2011):

> A definição original de usabilidade é de que os sistemas devem ser fáceis de usar e de aprender, flexíveis e devem despertar nas pessoas uma boa atitude. Além da usabilidade, a acessibilidade é hoje um dos principais objetivos do design, bem como a sustentabilidade. As metas da usabilidade hoje são vistas, essencialmente, como preocupadas com a eficiência e eficácia dos sistemas.

### Perguntas Iniciais sobre Usabilidade

Ao pensar em usabilidade, devemos fazer as seguintes perguntas:

- Esses produtos são realmente fáceis e agradáveis de utilizar?
- Em quantas ocasiões algum deles já causou algum incômodo?
- Ou levou mais tempo do que se previa para ser utilizado?

Apesar de as perguntas serem voltadas para produtos, são facilmente empregadas para o design de interfaces visuais.

### Acessibilidade versus Usabilidade

| Aspecto | Descrição |
|---------|-----------|
| **Acessibilidade** | Remoção de barreiras que, de outra forma, excluiriam totalmente algumas pessoas de usar o sistema |
| **Usabilidade** | Qualidade da interação em parâmetros como tempo consumido, número de erros e tempo necessário para se tornar um usuário competente |

**Hierarquia importante**: Um sistema deve ser acessível **antes de ser usável**. Um sistema pode ser avaliado como altamente usável, segundo algum critério, e mesmo assim fracassar se não for acessível ou não satisfizer as necessidades das pessoas.

### Considerações para Acessibilidade

Com um número cada vez maior de usuários e tecnologias diversas, os designers devem considerar as exigências que seus designs representam para a capacidade das pessoas. Isso inclui criar para idosos e crianças.

**Pontos essenciais:**
- Incluir pessoas com necessidades especiais na análise dos requisitos e testes dos sistemas
- Considerar se novas características afetarão os usuários com necessidades especiais (positivamente ou negativamente)
- Levar em consideração as diretrizes de acessibilidade
- Incluir usuários com necessidades especiais nos testes de usabilidade e testes beta

### Características de Alto Grau de Usabilidade

Um sistema com alto grau de usabilidade deve ser:

#### Eficiente
- As pessoas podem fazer coisas, interfaces e aparatos diversos mediante uma quantidade adequada de esforço

#### Eficaz
- Contém funções e conteúdo de informações adequadas
- Organizadas de forma apropriada

#### Fácil de Aprender
- As pessoas conseguem aprender como fazer as coisas
- O sistema é fácil de lembrar após algum tempo

#### Memorável
- Manter a atividade na memória é fundamental
- Exemplo: todos sabem onde clicar para salvar um arquivo no Microsoft Word

#### Seguro
- Seguro de operar na variedade de contextos em que será usado
- Alto grau de proteção contra erros e suas consequências

#### Útil
- Alto grau de utilidade no sentido de que fará as coisas que as pessoas querem que sejam feitas

#### Design Conceitual Claro
- Adotar um design conceitual claro e bem estruturado
- Fácil de comunicar às pessoas
- Design complexo torna o processo muito mais difícil
- Buscar um modelo conceitual claro, simples e consistente aumenta a usabilidade

### Affordance

#### Conceito

**Affordance** é uma palavra intraduzível para o português que passou a ser conhecida em sua língua original. 

Affordances são **características únicas dos objetos e dos ambientes** que dão suporte às suas contribuições para uma interação. O ambiente contém informação sobre a função dos objetos, na forma de affordances, ou seja, **a possibilidade de ação ofertada ao observador** por um detalhe no objeto ou no ambiente.

#### Importância do Affordance

- Ao perceber as affordances dos objetos, um observador é capaz de acessar a significação funcional que eles têm no ambiente
- **Perceber affordances é perceber como se pode interagir com o ambiente**
- Interagir com o ambiente digital, seja qual for, é testar sua boa usabilidade e seu bom design

## 4 Importância do Planejamento de Interface

Neste capítulo, você viu como é importante pensar em usabilidade e como é preciso planejar a interface, seja para:
- Seu app
- Jogo
- Site

**Objetivo final**: Proporcionar facilidade na utilização e prover uma melhor experiência ao jogador ou usuário.

### Reflexão Pessoal

**Pergunta para autoavaliação**: Supondo que você tenha um smartphone ou tablet, faça a seguinte pergunta para si mesmo:

> Quanto tempo eu demorei para aprender a usar e utilizar essa interface? 3 minutos? 5 minutos? 10 minutos?

Esta reflexão ilustra o poder de uma boa usabilidade e design intuitivo.

## Resumo de Pontos-Chave

1. **Wireframes** são ferramentas essenciais de prototipagem que precedem o design final
2. Profissionais de diferentes áreas utilizam wireframes para colaborar efetivamente
3. **Metáforas de interface** facilitam o aprendizado intuitivo, mas não devem ser limitantes
4. **Usabilidade** é fundamentalmente sobre criar sistemas eficientes e agradáveis
5. **Acessibilidade** deve preceder a usabilidade
6. **Affordance** permite que os usuários percebam intuitivamente como interagir com os elementos
7. O planejamento adequado de interfaces resulta em melhor experiência do usuário
